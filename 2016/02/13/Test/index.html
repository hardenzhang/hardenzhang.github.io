第二章 变量和基本类型

第二章的重点是 顶层const和底层const，以及与auto，decltype的联系

char可以是signed char，也可以是unsigned char，取决于编译器
表达式中不要混着用signed和unsigned，不然会出问题的

补码 = 负数 + 无符号数的模

浮点数运算用double

十进制字面值为int，long，long long中尺寸最小的一个
八，十六进制为int，unsigned int， … , long long, unsigned long long中尺寸最小的一个

转义序列：\1234 == \123 因为八进制最多三位

Variables must be defined exactly once but can be declared many times
we must declare the type of a variable before we can use that variable

Naming conventions are most useful when followed consistently

Define Variables Where You First Use Them

const默认文件内有效。若想用于文件之间，const定义和声明都要加extern

constexpr int == int const //指向整数的常量指针
constexpr的初始值必须为常量表达式或者constexpr函数

不能定义指向引用的指针，因为引用不是对象，没有地址。同理，不能定义指向引用的引用。

c++ 11中的初始化列表{ }的重要特性：用于built-in type时，如果初始值有丢失信息的风险，会报错。

复合类型（compound type） = base type + declarator
declarator包含变量，并指出变量和base type的关系

It may be helpful to think of pointers and references to const as pointers or references “that think they point or refer to const

顶层const和底层const*（top-level const, low-level const）
top-level const indicates that an object itself is const，顶层const修饰的是object！！
Low-level const appears in the base type of compound types such as pointers or references，底层const跟指针和引用有关！！
指针可以同时拥有顶层const和底层const

普通类型的const都是顶层const
const int &i = j; //const是底层const
const int * const i = &j; //左const是底层const，右const是顶层const
重点关注底层const，而顶层const可以忽略
When we copy an object, top-level consts are ignored

typedef char *pstring;
const pstring cstr = 0; //cstr是指向char的常量指针

auto
//auto用于一次声明或者定义多个变量时，基本数据类型（bulit-in type）要一样
int i = 0, &r = i;
auto a = r; //a是一个整数，因为编译器以引用对象的类型作为auto的类型
const int ci =i, &cr = ci;
auto b = cr; //b是一个整数，因为编译器以引用对象的类型作为auto的类型，且auto一般会忽略掉顶层const
auto c = &ci; //&ci是const int *类型，当中的const是底层const，不会被忽略
const auto c = ci; //f是cosnt int类型
auto &d = ci; //d是const int &类型
const auto &e = 42;
//auto中被忽略的可自己加上，比如引用和顶层const

decltype
const int ci = 0, &cj = ci, pk;
decltype(ci) a = 0; //a是const int类型
decltype(cj) b = x; //b是const int &类型，decltype括号内是一个变量，返回变量的类型（包括顶层const和引用）
decltype(cj + 0) c; //c是int类型，decltype括号内是一个表达式，返回表达式结果对应的类型
decltype(pk) d; //d是const int &类型，因为解引用指针会得到指针指向的对象，而且能给这个对象赋值
decltype( (ci) ) e; //错误，因为decltype( (variable) )的结果永远是引用，所以需要初始化！！
decltype用于函数时，返回函数类型。如果要使用函数指针类型，需要类似于decltype(compareIsbn) ，显示地加上
